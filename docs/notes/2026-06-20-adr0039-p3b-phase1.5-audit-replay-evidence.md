# ADR 0039 P3b Phase 1.5 — Production Audit Replay Evidence

本记录覆盖 P3b Phase 1.5 的隔离回放验证。范围限定为：从生产 `audit.jsonl` 读取真实 rules lane 历史审计行，映射成带明确 `historical_audit_backfill` provenance 的 Constraint Evidence Events，在临时 `ABRAIN_HOME` 中写入 L1 event，再用现有 shadow compiler 路径读取这些事件并生成 coverage 与 legacy delta。此阶段不打开 `ruleInjector.dualReadAudit.enabled`，不切换 compiled view injection，不进入 P4。

## 实现范围

新增 `scripts/dossier-constraint-audit-replay.mjs`，注册为 `dossier:constraint-audit-replay`。脚本默认使用临时 `ABRAIN_HOME`，拒绝非临时 `--abrain` 路径，输出 `.state/sediment/constraint-events/audit-replay/report.json` 与 `append-audit.jsonl`，并在未传 `--keep` 时清理临时目录。

新增 `extensions/sediment/constraint-evidence/audit-replay.ts`。映射表版本为 `constraint-audit-replay-mapping/v1`，支持 `create`、`update`、`merge`、`archive`、`reject` 五类 production rules audit operation，分别映射为 signal、correction、retract、rejection 类型 event。每个 event body 都包含 `replay_provenance.source=historical_audit_backfill`、审计文件哈希、审计行号、审计 operation、replay run id、harness version、mapping table version 与 mapping table hash。

Shadow compiler 事件扫描与 coverage 报告已扩展：`event-scan.ts` 将完整 `replay_provenance` 映射到 `ConstraintEventSourceRecord.replayProvenance`；`event-report.ts` 在 `event-coverage.json` 中输出 provenance 统计与每行 replay provenance；`normalize.ts` 将 replay provenance 纳入归一化输入，避免 replay 来源、审计行、harness version 或 mapping table 变化时 input root hash 静默不变。

## Shadow Compiler 证据

将隔离回放目录接入现有 shadow compiler：

`node scripts/dossier-constraint-shadow-report.mjs --force --write --abrain /tmp/constraint-audit-replay-5xOUKI --include-projects none --project pi-global --model openai/gpt-5.5 --max-retries 1`

首次运行暴露了 prompt 约束缺口：模型在 `diagnostics.data.operation` 中输出了被 validator 禁止的 mutation-key 字段，导致 `SC_COMPILER_VALIDATION_FAILED`。未放松 validator；修正方式是在 prompt 中明确禁止任何输出对象包含 mutation-key 字段，包括 `diagnostics.data`。修正后重跑成功。

成功运行结果：`ok=true`，`inputRootHash=3d645f4b32d871e30901ef93db850c30471fb876c3b621d4e89d34f553bdc4c8`，`validationHash=281217f2af2ee7eb1291c217aba03846d77dd353d3bf27297f3c500f649d4090`，`sourceCount=10`，`constraints=5`，`exclusions=5`，`unresolved=0`，`unmappedSources=0`，`shadowOutputHash=396067e69fa13272bd13880898585379458ab172cf2495c128596ee0b2192e72`，`rulesFileListChanged=false`。本次 diagnostics summary 为 `SC_UNCLASSIFIED=2` 与 `SC_LEGACY_PARALLEL_DELTA=3`。

生成的 `/tmp/constraint-audit-replay-5xOUKI/.state/sediment/constraint-shadow/latest/event-coverage.json` summary 为 `totalEvents=10`、`validEvents=10`、`invalidEvents=0`、`queuedEvents=0`、`projectedEvents=10`、`staleEvents=0`、`appendFailedEvents=0`、`coverageRatio=1`。provenance summary 为 `liveEvents=0`、`replayBackfillEvents=10`、`manualEvents=0`、`unknownEvents=0`。每行均带 `sourceChannel=replay` 与 `provenance.source=historical_audit_backfill`。

## 边界结论

Phase 1.5 已证明：生产 rules audit 行可以通过隔离 harness 转换为内容寻址 L1 Constraint Evidence Events；event hash/path/envelope 校验通过；mapping provenance 可机器检查；现有 shadow compiler 可以读取这批 replay events 并生成 event coverage 与 legacy delta；canonical `rules/`、`knowledge/`、`projects/` 未被写入。

本阶段仍不授权打开 `ruleInjector.dualReadAudit.enabled=true`。下一步如要进入 dual-read runtime observation，必须基于本 evidence package 再做一次多模型复审，并且继续保持 P3 compiled-view injection 与 P4 关闭。
