# ADR0040 D3-v2 session_start single-consumer execution-ready (S1 / R3.9)

This note freezes the pre-activation boundary for the first D3-v2 `session_start` consumer after the R3.9 live-selector no-trim residual repair (on top of R3.8 rehearse stateRoot containment + fail-closed + session_id/path-component length-split). It does **not** authorize S2 activation, does **not** create or bind a real session, and does **not** enable any live selector.

## Architecture (R3.9)

- Independent versioned adapter under `extensions/_shared/proposition-lifecycle-freshness-d3-v2-session-start.ts` plus closed helpers:
  - `...-activation.ts` — controlled-root activation object loader (absolute path, NOFOLLOW/identity walk, exact JCS+LF, self-hash without `activation_object_hash`); exports pure `validateBoundActivationObjectClosed` (all BOUND_KEYS required, no trimmed objects, no production I/O) and `activationRootHasNoBoundObject` (extension-agnostic NOFOLLOW size-limited scan of dedicated activation root; any readdir/lstat/read/parse error fail-closed false; any AUTHORIZED/bound/executable file makes predicate false; never scans outside root). `settings_mutation` is a closed-schema set exact-equal to the live norm object (includes `expectedIntentHash`, `maxReadBytes`, `selector`). **R3.7/R3.8**: builder + full validator force `session_id` to a **single safe filename component** — non-empty, length ≤ **128**, ASCII `[A-Za-z0-9._-]` only; reject `'.'`/`'..'`, any `/` `\`, NUL, control chars. Generic path components used by `joinUnderRootContained` allow ≤ **Linux NAME_MAX 255** so derived names like `${sessionId}.json` / `${sessionId}.jsonl` at the 128-char session-id boundary remain legal. `activation_nonce` remains lowercase SHA-256 hex (strict subset of the safe charset). Exports `assertSafeSessionIdComponent`, `isSafeSessionIdComponent` (non-throwing predicate), `assertSafeActivationNonceComponent`, `assertSafeSinglePathComponent` (default max 255), `joinUnderRootContained`, `assertResolvedPathContainedUnderRoot`. Selector `session_ids[]` reuse the same safe-component rule (≤128).
  - `...-fence.ts` — balanced/nested managed BEGIN..END parser; sanitizer removes only outermost managed regions and never mutates any byte outside those regions
  - `...-rollback.ts` — halt/taint/**pending-intent** consumption, real session quarantine rename, intent→action→postcondition→receipt FSM, triple-authorization production gate with exact path bind; Door1 **reuses** the activation-module closed-schema validator (does not copy a weaker key/selfhash-only check; inherits R3.7 session_id safety). R3.5 path preflight retained: before any mkdir/settings/intents write, settings parent chain, stateRoot chain, session/quarantine parent chains, and activation targets are walked with lstat NOFOLLOW; existing ancestors must not be symlinks and existing intermediates must be regular directories; sandbox then requires realpath not land under any hard production root. **R3.6 closes the check-after window**: all create/write paths use a **Linux retained parent-directory FD walk** starting at `/` with an open directory FD; each next component is `lstat`/`open`/`mkdir`'d **only** through `/proc/self/fd/<parentFd>/<basename>`; previous layer FD released only after next layer FD is open; any symlink refused; target parent FD kept open for the whole create/write critical section. `ensureDirectoryChainNoSymlink` never falls back to absolute-path mkdir after a check. `atomicDurableWriteText/Json` create tmp, fsync, rename-to-final, and fsync-parent only through the anchored parent FD. Quarantine rename holds **both** source-parent and dest-parent FDs and performs final re-lstat + rename on procfd relative paths (RENAME_NOREPLACE dest-race residual remains honest). Settings read/write/postcondition use the same anchored open. Non-Linux or missing `/proc/self/fd` **fail-closed**. This is a **Linux boundary**, not a portable claim. **R3.7 path helpers**: `haltMarkerPath` / `sessionTaintPath` / `rollbackIntentDir` / `rollbackReceiptDir` / `rollbackFaceArtifactPath` / `rollbackBarrierPath` / `haltFailurePath` all validate safe basename + `joinUnderRootContained` so a hostile `session_id` cannot escape `stateRoot`. **R3.8 rehearse containment**: public `rehearseD3V2SessionStartRollback` forces `sandboxSettingsPath` / `sessionFilePath` / `quarantineTarget` **and their parents** to be strictly contained under the unique disposable `path.resolve(stateRoot)` **before any mkdir/write**. External absolute, sibling, and `..` overrides are rejected (pure string resolve; no I/O). Defaults remain `joinUnderRootContained` under stateRoot; optional overrides are retained but contained. Main operator + public rehearse writes are procfd-anchored; test fixture setup lives only in smoke scripts. Closed-set R3.6 ancestor-swap test hook residual (absolute rename of the sandbox ancestor itself) remains accepted.
- Control-flow entry: `extensions/abrain/rule-injector/proposition-lifecycle-freshness-d3-v2-session-start-control.ts`
- Host wiring predicate: `evaluateD3V2SessionStartHostWiringPredicate` verifies `extensions/abrain/index.ts` imports and calls `activateRuleInjector(pi)` and that rule-injector registers `session_start`/`before_agent_start` into `decideD3V2SessionStartControl`
- Settings key `ruleInjector.propositionLifecycleFreshnessD3V2SessionStartInjection` is schema-default `enabled=false`. When `enabled=true`, schema and runtime both require `activationObjectPath` + `activationObjectHash` (plus D3/adapter pins). Live settings intentionally leave the key absent so production remains inert.
- **R3.8/R3.9 live selector**: schema `selector.session_ids` items require `maxLength: 128`, ASCII pattern `^[A-Za-z0-9._-]+$`, and `not enum [".",".."]` (schema already rejects whitespace; runtime stays consistent). Runtime `normalizeLiveSelectorSessionIds` / `resolveD3V2SessionStartInjectionSettings` apply `isSafeSessionIdComponent` to each **original** item with **no trim**: any non-string, empty, pure-whitespace, leading/trailing-whitespace, null, or otherwise unsafe entry **fail-closes** to `enabled=false` + empty selector + cleared activation pins **without throw** (including no missing-pin throw). Empty entries are **not** ignored. `selectD3V2SessionStartSession` safety-checks the raw `getSessionId()` value with **no trim / no identity rewrite**; empty / whitespace / padded / unsafe forms are unselected (no D3, no inject); safe IDs keep original form for selector equality.
- Bound activation requires `mode=bound`, `authorization_status=AUTHORIZED`, `executable=true`, matching safe `session_id`, unique `activation_nonce`, D3 identities, adapter manifest, closed settings mutation, audit/rollback targets, session file path/dev/ino + activation-time prefix, and absent quarantine target. Activation objects must not self-reference `settings.activationObjectHash`; runtime separately verifies the settings pin equals the object self-hash.
- Selected path: validate activation+manifest → halt/taint/**pending intent** check → D3 read → construct expected exact fence → exact own is idempotent after successful audit; foreign/mixed/malformed are sanitized from in-memory systemPrompt only (no session write, no taint/rollback) then inject v2 on success or return sanitized zero-rule prompt on failure. Never keep v1/compiled/legacy fences. `selected_zero_injection` returns sanitized `systemPrompt` when present.
- Fence / audit / `cachedRules` / `getCurrentRuleInjectionNonce` share the bound activation nonce. `context-packer` and `llm-extractor` (normal window + continuation provider message text parts) call the same strip; only the selected activation is removed; unselected text is byte-identical.
- Runtime audit is **v2-only**: under exclusive lock the entire ≤8MiB file is validated (exact JCS+LF per line, schema/version, zero-based real `pre_offset`, first `parent_hash=null`, subsequent parent=self, self-hash). v1 / offset gap / tail truncate / unknown fail closed. Append fsyncs file+dir before prompt mutation is authorized. Audit v1 compatibility helpers are removed.
- Rollback faces: `selector_disable` → `session_taint` → `session_quarantine_rename` → `terminal_halt`. Each face writes durable intent via atomic temp+fsync+rename+dir fsync (R3.6: all via retained parent FD), executes, verifies postcondition, then durable receipt the same way. Intent reads require exact JCS+LF/schema/selfhash/nonce/session/face **and** `parent_hash ===` current receipt-chain `parentHash` (stale/out-of-order intents fail closed). Intent without receipt resumes only with `operatorContinue=true`. Quarantine resume re-verifies dest dev/ino/prefix against activation binding via retained parent FD. Any pending intent is treated as halt by the selected runtime (closes selector-disable crash injection window).
- **stateRoot identity bind**: `path.resolve(stateRoot) === path.resolve(activation.rollback_target)` is mandatory; production-target auth `production_state_root` binds the same path. Settings/session/quarantine exact binds remain.
- Production default-deny is exact: **missing any of the three** mutually bound AUTHORIZED doors (bound activation closed-schema selfhash + independent rollback auth selfhash + independent production-target auth selfhash) fails closed. Production-target auth exact-binds `production_settings_path` / `production_state_root` / `production_session_file_path` / `production_quarantine_target`. After all three doors, only those exact bound paths may execute. Args may not override activation session/quarantine binds.
- Sandbox hard-path roots are derived from `HOME` + default activation root (boundary-safe exact/`${root}${sep}` prefix): `~/.abrain`, `~/.pi/agent` (settings/sessions), `~/.pi/.pi-astack/adr0040-d3-v2-session-start` control subtree, runtime-audit file, default activation root. **No substring exemptions.** Sandbox `stateRoot` pointing at the real control root is refused before any write; pre/post identity must be unchanged. R3.5 additionally refuses ancestor-symlink aliases under `/tmp` that realpath into those hard roots (control root and agent) before any production byte/directory change. R3.6 additionally makes post-preflight ancestor swaps unable to redirect writes into production: retained parent FDs pin the original directory inode.
- Quarantine is a **real rename** (not copy+delete). Source FD **and** source/dest parent FDs are held across the critical section; source identity is re-lstat'd via procfd immediately before rename; dest must be NOFOLLOW-absent via dest-parent procfd. Node `fs.renameSync` cannot express `RENAME_NOREPLACE`; residual same-machine non-cooperative dest-race overwrite risk is documented honestly — we do **not** claim full no-overwrite atomicity. Concurrent dest creation that leaves postcondition unsatisfied halts (no silent success). Ancestor-swap rewrite of the rename target is closed by retained parent FDs.
- Receipts are **face-named fixed-path self-hashed** artifacts (JCS body + self-hash). They are **not** content-addressed storage.
- Adapter manifest critical exact-byte set covers adapter/control/audit/index, **`extensions/abrain/index.ts` host entry**, activation/fence/rollback helpers, `pi-internals.ts`, `pi-astack-settings.schema.json`, `package.json`, `package-lock.json`, `context-packer.ts`, `llm-extractor.ts`. Validator fail-closes on any missing surface. Dead `RULE_FENCE_RE` removed from rule-injector index.
- **R3.6/R3.7/R3.8 test surface (closed-set, never production-default)**:
  - Low-level helpers exported for unit tests: `walkRetainParentDirectoryFd`, `procFdChildPath`, `ensureDirectoryChainNoSymlink` (procfd-based), `assertSafeSessionIdComponent`, `isSafeSessionIdComponent`, `joinUnderRootContained`, `assertResolvedPathContainedUnderRoot`, path helpers.
  - Optional operator arg `__testAncestorSwapAfterPreflight` of kind `r3.6_sandbox_ancestor_swap_after_preflight_v1`: requires `target===sandbox`, non-empty env `PI_ASTACK_R36_TEST_TOKEN` exact match, sandbox ancestor under system temp, hard-root target exact-listed, one-shot per process. Applied after preflight and before any mkdir/open/rename. Production target refuses the hook. Tests snapshot production roots only — never write into real root content.
  - R3.7 smoke: adversarial `sessionId='../...'` / absolute / backslash / dot rejection at builder, full validator, and path helpers; containment of taint/intent/receipt/halt under stateRoot; static assertion that `rehearseD3V2SessionStartRollback` has no absolute write/exists/lstat/read; live rehearse via anchored create.
  - R3.8 smoke: rehearse external absolute / sibling / `..` settings + external session/quarantine overrides refused before any mkdir/write with dynamic external sentinel unchanged; live resolve unsafe selector fail-closed disabled/empty without throw; unsafe current session id leaves control unselected (no D3/no inject); session_id 127/128 pass and 129 refuse; generic component 255 pass / 256 refuse; `${id}.json/.jsonl` at 128 boundary pass under `joinUnderRootContained`; schema items maxLength 128 + ASCII + reject `./..`.
  - R3.9 smoke: empty / pure-whitespace / leading-trailing-whitespace / null selector entries fail-closed disabled/empty/cleared pins without missing-pin throw and without trim-to-safe rewrite; select refuses raw empty/whitespace/padded/unsafe session ids as unselected (identity not rewritten); safe ids keep original form.
- S1 dossier emits a `NOT_AUTHORIZED` / `executable=false` activation **template** only (candidate null unless explicit CLI). Assertions are real file/source/field predicates (including Door1 reuse, intent parent enforcement, hard-path coverage, quarantine residual honesty, R3.5 path-preflight source predicates, R3.6 retained-parent-fd predicates, R3.7 session_id safety + path containment + rehearse anchored write, R3.8 rehearse stateRoot containment + live selector fail-closed + length split + schema selector items, R3.9 no-trim raw selector/session fail-closed, activation-root fail-closed scan). Zero-write **snapshot** covers only this task's explicit production roots — activation root / runtime audit / rollback root (non-existence is valid prestate). The live `~/.pi/agent/sessions` tree is **not** content-snapshotted: concurrent non-ADR session appends are legitimate background state and must not make strict `--verify` drift. Live-sessions zero-write for this round is proven by candidate unresolved + `candidate_session_not_created` + `production_bound_activation_not_created` (extension-agnostic recursive NOFOLLOW nested-JSON scan: any AUTHORIZED/bound/executable is false; parse/read errors fail-closed false) + read-only dossier mode. `--verify` remains strict byte-for-byte (no prestate soft-pass). ## Linux boundary (R3.6)

Retained parent-directory FD walks require Linux `/proc/self/fd`. Non-Linux platforms and inaccessible `/proc/self/fd` fail closed with `procfd_unavailable`. This is an intentional platform boundary, not a silent portable fallback.

## Accepted residual (unchanged)

- Node `fs.renameSync` has no `RENAME_NOREPLACE`; same-machine non-cooperative dest-race overwrite residual on quarantine rename remains honest.
- Closed-set R3.6 test hook may absolute-rename a sandbox ancestor for deterministic testing (never on production default path).

## S2 / P3 remain blocked

The dossier binds exact production D3 hashes and an activation-object template, but does **not** create a persisted session, does **not** write a bound activation object, and does **not** enable any live selector. Fresh user authorization is still required before any S2 atomic create/bind unit. Production rollback against real hard paths requires a separate fresh triple authorization and is default-deny without it (missing any of the three doors); this turn never invokes that path. S2 and aggregate P3 are **not** authorized by this note.
