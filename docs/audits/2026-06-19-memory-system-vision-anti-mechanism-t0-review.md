# Memory System Vision 反机械主义 T0 讨论记录

> 本文件记录第二大脑记忆系统愿景文档的反机械主义讨论。讨论要求：T0 互相回应并达成一致；主会话只转述、记录和整理文本，不裁决。

## 输入

- `docs/memory-system-vision.md`
- 用户要求：讨论重点是反机械主义；T0 必须互相讨论达成一致，不能由主会话裁决。

## R1

四个 T0 均为 REVISE。共同判断：愿景目标正确，但缺少显式反机械主义原则。风险集中在：记忆类别变成 schema 槽位，当前/历史变成状态字段，强弱信号变成权重公式，演化操作变成 API / 状态机，隐私边界变成权限等级。

关键定义收敛为：机械主义是把本应在使用当下、基于证据与语境重新做出的判断，预先冻结为标量、状态字段、固定分类或自动推进规则，并在未来机械套用。

## R2

四个 T0 均同意需要写入反机械主义原则，但存在三项差异：

- Git 是唯一底座，还是默认 / 首要底座并允许敏感记忆物理隔离。
- 词汇约定是否写入，以及如何避免变成机械清单。
- 是完整改写全文，还是新增统御节并最小修订相关段落。

## R3

四个 T0 全部 SIGN 以下方向：

- 使用“语义判断保留在使用时刻的认知层，不固化为存储层的标量、状态字段或自动推进规则”作为单一上位原则。
- 反模式是示例，不能变成合规检查清单。
- 索引、搜索、候选集缩小等基础设施可以存在，但只能帮助找到证据、降低成本、保持可追溯；不得替代语义判断。
- 当前 / 历史、强 / 弱、精确 / 模糊、提升 / 降级等词是解释语言，不是存储字段、枚举状态或自动门禁。
- Git 改为非敏感持久记忆的默认 / 首要底座；敏感内容可以不进入 Git，或采用同构但物理隔离的版本化存储；隐私边界由自然语言定义。
- 不替换全文；新增“反机械主义原则”统御全文，并修订 Git 段落，词汇约定作为解释性附录写入。

## R4 ratify

R3 后，主会话整理出一份待 ratify 文本。该文本不得视为主会话裁决，需由 T0 对同一文本继续 SIGN / REVISE。

R4 结果为 4/4 SIGN，无阻断项。非阻断建议包括：收紧“隐私边界等硬约束”的开放式措辞、将“追踪错误来源”改为“追溯内容变更来源”、明确用语约定不是合规检查清单。落盘时已采纳这些非阻断措辞。
