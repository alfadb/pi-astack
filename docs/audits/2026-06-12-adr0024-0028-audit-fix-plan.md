1. **target_entry_slug 不可能指向 rule**:classifier 的 related_entries 来自 memory_search corpus,而 `rules/` 被排除在 corpus 外(parser.ts:33 `WORLD_EXTRA_IGNORE_DIRS`)。"改某条规则"场景下 classifier 找不到 target → 走 no-target Tier-1 create → writeAbrainRule 的 Jaccard 闸 → adjudicator(lane ON)裁决 {update,merge,create}。**规则更新场景已被 PR-4 机制覆盖**(F7 flip 后完整生效)。

### A.2 设计:`!target` 从排除合取改为"仅限制 conf≥8 过渡 fallback"

新谓词:

```
isTier1Directive = signal_found ∧ durable ∧ provenance=user-expressed
                   ∧ ( is_directive === true                       ← 不再要求 !target
                     ∨ (conf ≥ 8 ∧ !target_entry_slug) )           ← 过渡 fallback 保持原状,随 SUNSET 消亡
```

**路由语义**(两权威仍 disjoint,R1' 不变):
- **指令本体 → classifier 权威,确定性提交为规则**:带 target 的 is_directive 信号同样走 tier1_direct create(buildTier1RuleDraft 不变)。若与既有规则近重 → 既有 Jaccard 闸/adjudicator 处理(路径复用,零新机制)。指令从此**永不可被 LLM skip**(R2')。
- **被指向的知识条目 → curator 权威,Tier-2 语义不变**:条目的 update/supersede/archive 由 curator 在同窗 follow-up(PR-A2 机制)中带着 correction context 完成;curator 对**条目**的 skip 仍合法(指令已另行确定性捕获,不构成丢失)。

**配套修改(opus C1 规格化)**:PR-A2 follow-up 递归改为 `{ ...args, tier1ExtractorFollowUp: false, tier1AlreadyCommitted: true }`(**保留 correctionSignal**);tier1 块条件改 `tier1Signal && !args.tier1AlreadyCommitted && shouldEscalateToCurator(...)`。门控是**主**去重器,body-hash 是**备**;不变量:follow-up 重入绝不产生第二条 `tier1_direct_write` audit 行(smoke 断言)。curator 照常收到含 `correction_intent`/`most_likely_error` 的完整 CorrectionSignal 作为条目衰变 context,引导其对 stale 条目 supersede/update 而非再造重复。

**R2' justification(deepseek #2)**:R2' 的封闭空间约束作用于 Tier-1 路径内的 router/curator(处理指令本体);被指向条目的衰变是 Tier-2 领地。curator 对条目的 skip 可接受的两个结构性理由:(1) rules 每轮强制注入,读侧 LLM 跟随最新规则而非 stale entry;(2) AX-PROVENANCE 轴(user-expressed rule vs assistant-observed entry)为将来读侧降权提供结构支撑。可观测性:tier1 audit 行记 `target_entry_touched`(follow-up 结果是否触及 target slug),让"curator 对 targeted 条目持续 skip"可被 aggregator 观察。
